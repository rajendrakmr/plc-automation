## 🧭 Overview

**DevOps Monitoring & AWS Cost Analysis API** is a production-ready internal utility service that gives DevOps and development teams a single REST API for:

- 📊 **Real-time infrastructure monitoring** — CPU, memory, and disk metrics collected via `psutil` with configurable alert thresholds
- 📋 **Application log analysis** — ingest and auto-classify logs (INFO / WARNING / ERROR) for faster debugging
- 💰 **AWS cost tracking** — automated cost reports and resource usage analysis using AWS Cost Explorer and S3 via `boto3`

Built with a clean, modular FastAPI architecture (`routers/`, `services/`, `app/`), this API improved AWS cost visibility and reporting efficiency by **60%** for development teams and enhanced production server CPU health tracking across multiple environments.

## 🏗 Architecture

```
HTTP Request
     │
     ▼
┌─────────────────────────────────────────────────┐
│                   main.py                        │
│      FastAPI app + router registration           │
└──────────────────┬──────────────────────────────┘
                   │
       ┌───────────┼────────────┐
       ▼           ▼            ▼
┌───────────┐ ┌─────────┐ ┌──────────┐
│ routers/  │ │routers/ │ │routers/  │
│ system    │ │  logs   │ │   aws    │
└─────┬─────┘ └────┬────┘ └────┬─────┘
      │             │           │
      ▼             ▼           ▼
┌───────────┐ ┌─────────┐ ┌──────────────────┐
│ services/ │ │services/│ │    services/      │
│  psutil   │ │  log    │ │  boto3            │
│ CPU/Mem/  │ │ parser  │ │  Cost Explorer    │
│ Disk      │ │         │ │  + S3             │
└───────────┘ └─────────┘ └──────────────────┘
```

### Module Responsibilities

| Layer | Folder | Responsibility |
|---|---|---|
| Entry Point | `main.py` | FastAPI app instance, router registration, middleware |
| Routes | `routers/` | Define HTTP endpoints, request/response models |
| Business Logic | `services/` | psutil metrics, log parsing, boto3 AWS API calls |
| App Config | `app/` | Settings, config loading, shared utilities |

## 🔌 API Endpoints

### 🖥 System Monitoring

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/system/health` | Overall system health status |
| `GET` | `/system/cpu` | Real-time CPU usage with alert threshold check |
| `GET` | `/system/memory` | Memory — total, used, free, percentage |
| `GET` | `/system/disk` | Disk usage per mount point |
| `GET` | `/system/metrics` | Full snapshot — CPU + memory + disk combined |

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/logs/ingest` | Ingest application logs for analysis |

| `GET` | `/logs/warnings` | Fetch WARNING-level log entries |

| `GET` | `/aws/s3/usage` | S3 bucket resource usage report |
| `GET` | `/aws/s3/buckets` | List S3 buckets with size and object count |

## ⚡ Getting Started

### Prerequisites

- Python 3.11+
- pip
- Docker (for containerized run)
- AWS account with IAM credentials (for cost & S3 features)

---

### 1. Run Locally

```bash
# Clone the repository
git clone https://github.com/rajendrakmr/devops-utilities-api.git
cd devops-utilities-api

# Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate         # Linux/Mac
# venv\Scripts\activate           # Windows

# Install dependencies
pip install -r requirements.txt

# Configure environment variables
cp .env.example .env
# Edit .env with your AWS credentials and thresholds

# Start the server
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

Visit: **`http://localhost:8000/docs`**

---

### 2. Run with Docker

```bash
# Build the Docker image
docker build -t devops-utilities-api .

# Run the container
docker run -d \
  --name devops-api \
  -p 8000:8000 \
  -e AWS_ACCESS_KEY_ID=your_key \
  -e AWS_SECRET_ACCESS_KEY=your_secret \
  -e AWS_DEFAULT_REGION=us-east-1 \
  devops-utilities-api

# Verify the container is running
curl http://localhost:8000/health
```

Visit: **`http://localhost:8000/docs`**

### 3. Docker Image Variants

This project ships with **three Dockerfile variants** for different environments:

| Dockerfile | Tag | Use Case |

## 🔐 AWS Configuration

The API uses **boto3** to connect to AWS Cost Explorer and S3. Create an IAM user with the following policy:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "ce:GetCostAndUsage",
        "ce:GetCostForecast",
        "ce:GetDimensionValues",
        "s3:ListAllMyBuckets",
        "s3:GetBucketLocation",
        "s3:ListBucket"
      ],
      "Resource": "*"
    }
  ]
}
```

**Configure credentials (choose one):**

```bash
# Option 1: AWS CLI (recommended for local dev)
aws configure

# Option 2: Environment variables
export AWS_ACCESS_KEY_ID=your_access_key
export AWS_SECRET_ACCESS_KEY=your_secret_key
export AWS_DEFAULT_REGION=us-east-1
```
